# Fix Review Comments

You are about to fix review comments posted for a GitHub PR.

## Instructions

1. Read the comments provided by the user (they were generated by the PR topic review GUI).
2. Each comment is grouped by topic and may include a file path and line number.
3. For each comment:
   - Inspect the referenced file and line.
   - Understand the concern.
   - Make the minimal fix that addresses the feedback.
   - Do not refactor unrelated code.
4. After all fixes, run the project's test command.
5. Summarize what you changed for each comment.

## Rules

- Do not resolve or dismiss GitHub comments unless the user explicitly asks.
- Do not push or merge without user confirmation.
- If a comment is unclear, ask the user before making changes.
- Keep fixes focused on the specific concern raised.
